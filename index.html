Day 1 · Fundamentals
Basics of System Design

Today I started reading the basics of System Design. 🚀 Initially, it felt overwhelming, but I realized it's actually built on simple concepts. At its core, every system has three main parts:

User → Server → Database

The user sends a request, the server processes it, and the database stores or retrieves data. But things get interesting when scale increases. If millions of users hit a single server, it can crash. That's where load balancing and horizontal scaling come in.

A simple analogy: A server is like a restaurant kitchen. If too many orders come in, one chef can't handle it. So we add more chefs (servers) and distribute the workload.

Big learning today: System design is not about complex tools — it's about solving simple problems at scale. 🚀

Day 2 · Infrastructure
Load Balancer

🚀 Today I learned about Load Balancers. Imagine thousands of users trying to access a website at the same time. If all requests go to a single server, it can easily crash. That's where a Load Balancer comes in.

User → Load Balancer → Multiple Servers

A simple analogy: Think of a load balancer as a receptionist in an office. When people arrive, the receptionist directs them to available staff instead of overloading one person.

This improves:
✅ Performance
✅ Availability
✅ Reliability

Some common strategies:

  • Round Robin (one-by-one distribution)
  • Least Connections (send to least busy server)

💡 Key takeaway: A load balancer ensures no single server becomes a bottleneck.

Day 3 · Scalability
Scaling — Horizontal vs Vertical

🚀 Today I learned about Horizontal vs Vertical Scaling. As systems grow, handling more users becomes a challenge. There are two ways to scale:

🔹 Vertical Scaling (Scale Up)
Increase the power of a single server (more CPU, RAM)

🔹 Horizontal Scaling (Scale Out)
Add more servers and distribute the load

User → Load Balancer → Multiple Servers

A simple analogy:
Vertical scaling = upgrading your bike to a faster one.
Horizontal scaling = adding more bikes for more deliveries.

✅ Handles massive traffic
✅ Improves availability
✅ Avoids single point of failure

💡 Key takeaway: Scaling is not just about power — it's about distributing load efficiently.
