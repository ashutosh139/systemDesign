Day 2: Load Balancer

🚀 Day 2 of Learning System Design Today I learned about Load Balancers. Imagine thousands of users trying to access a website at the same time. If all requests go to a single server, it can easily crash. That’s where a Load Balancer comes in. Instead of sending all traffic to one server, it distributes requests across multiple servers: User → Load Balancer → Multiple Servers A simple analogy: Think of a load balancer as a receptionist in an office. When people arrive, the receptionist directs them to available staff instead of overloading one person. This improves: ✅ Performance ✅ Availability ✅ Reliability Some common strategies: Round Robin (one-by-one distribution) Least Connections (send to least busy server) 💡 Key takeaway: A load balancer ensures no single server becomes a bottleneck.
