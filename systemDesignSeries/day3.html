Day 3: Scaling

🚀 Day 3 of Learning System Design Today I learned about Horizontal vs Vertical Scaling. As systems grow, handling more users becomes a challenge. There are two ways to scale: 🔹 Vertical Scaling (Scale Up) Increase the power of a single server (more CPU, RAM) 🔹 Horizontal Scaling (Scale Out) Add more servers and distribute the load Example: User → Load Balancer → Multiple Servers A simple analogy: Vertical scaling is like upgrading your bike to a faster one. Horizontal scaling is like adding more bikes to handle more deliveries. In real-world systems like Amazon or Flipkart, horizontal scaling is preferred because: ✅ It handles massive traffic ✅ It improves availability ✅ It avoids single point of failure 💡 Key takeaway: Scaling is not just about power—it’s about distributing load efficiently.
